Akademiya Office Suite 기능을 추가해줘.
1. Project Concept: Google Docs/Slides/Sheets와 같은 Web기반 오피스.
  1-1. 공통: Google Docs처럼 협업(동시 수정)이 가능해야 함. 처음 생성 시 조직 단위/반 단위/개인 단위로 문서를 생성할 수 있게 하고, 각 개인 하나에 최대 용량은 0.1GB(100MB)를 기본으로 할당해줄 것. 이 최대 용량은 MB 단위로 DB에 기록해둘 것. (100MB 하드코딩 금지) Google Docs처럼 권한 관리가 가능해야 함. (이메일로 제한/반의 모든 사용자/조직의 모든 사용자/링크가 있는 모든 사용자 단위로 뷰어/댓글 작성자/편집자) 당연히 문서 편집자 권한을 가진 사람은 문서 삭제도 가능해야 함. 폰트는 기본으로 Noto Sans/Serif CJK KR/JP/SC 및 Noto Sans/Serif 기본을 제공하고, 사용자의 언어 상황에 맞게 기본값을 정할것. (한국어 사용자는 Noto Sans CJK KR가 기본값, 영어 사용자는 Noto Sans가 기본값 등), 외부 폰트는 사용자가 직접 TTF/OTF를 업로드하여 사용할 수 있게 할 것. Akademiya Cloud (cloud.akademiya.kr)에서 자신이 작성한 Script/Beam/Grid 문서와 업로드한 .TTF/OTF와 이미지 파일, 그리고 조직/반 창에서는 조직/반 공유 단위의 문서를 볼 수 있게 해줘. Cloud 기능은 OCI Object Storage를 이용할 것. (dotenv 필수) 모든 UI 테마는 Akademiya 본 사이트와 통일하며, 대신 배치는 Google Docs와 같이 좌상단 Akademiya 로고, 문서 제목을 놓고, 그 오른쪽으로 쭉 MS Office와 같은 리본 메뉴를 사용할 것. 스마트폰 접속 시에는 문서 뷰어로만 작동할 것 (편집 불가능) 다만 태블릿은 Desktop과 동일하게
  1-2. Akademiya Script (office.akademiya.kr/script): Google Docs와 같은 웹 워드. 파일 형식은 ODT를 사용하고, 다운로드 목록에서는 .ODT로 내보내기, .DOCX로 내보내기, .PDF로 내보내기를 지원할 것. MS Word/Google Docs와 같이 다양한 서식 기능을 제공해야 함.
  1-3. Akademiya Beam (office.akademiya.kr/beam): Google Slides와 같은 웹 PPT. 파일 형식은 ODP를 사용하고, 다운로드 목록에서는 .ODP로 내보내기, .PPTX로 내보내기, .PDF로 내보내기를 지원할 것. MS PowerPoint/Google Slides와 같이 기본적인 서식과 표준 애니메이션을 지원할 것. 테마는 Basic Light, Basic Dark, Akademiya Light, Akademiya Dark, GMCAuto Light, GMCAuto Dark 이렇게 6개 지원할 것. (Basic 시리즈는 MS PPT/Google Slides 기본 테마 같은 느낌, Akademiya 시리즈는 Akademiya의 테마를 참고할 것, GMCAuto 시리즈는 GMCAuto의 테마를 참고할 것)
  1-4. Akademiya Grid (office.akademiya.kr/grid): Google Sheets와 같은 웹 스프레드시트. 파일 형식은 ODS를 사용하고, 다운로드 목록에서는 .ODS로 내보내기, .XLSX로 내보내기, .CSV로 내보내기를 지원할 것. MS Excel/Google Sheets에서 지원하는 함수와 페이지 기능들을 지원할 것. 

2. 구현 주의 사항
  2-1. 모든 스택은 Akademiya 본 사이트와 동일하게 할 것 (Node.js 24.16.0, React 19, Vite 8 등 버전 맞추기)
  2-2. Akademiya 본 사이트와 분리되게 구성할 것 (GMCAuto처럼 별도의 서비스)
  2-3. Akademiya 본 사이트에 Akademiya Cloud로 연결되는 링크를 좌측 메뉴 하단에 걸어둘 것 (GMCAuto 연결 링크와 같은 방식, 다만 HAFS한정 말고 전체 유저에게)
  2-4. OCI Object Storage API 호출 건수를 최소화하기 위해 편집자 중 단 한 명이라도 편집을 시작했을 때 서버의 임시 저장 공간에 불러올 것. 활동이 있을 시 5분마다 Object Storage에 자동 저장하며, 15분간 활동이 없을 시 Object Storage에 저장 후 임시 파일을 삭제할 것.
